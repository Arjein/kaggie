üêõ Problem Description

Previously, when users entered their API keys and saved them in the settings modal, the warning banner that says "API keys required - use the settings button in the navbar to configure" would not disappear immediately.

API keys required - use the settings button in the navbar to configure

üîç Root Cause Analysis

The issue was caused by two separate instances of the useApiKeys hook:

  • App.tsx: Used to display/hide the warning banner
  • Navbar.tsx: Used in settings to check API key status

When API keys were saved, the navbar's hook was refreshed but the App component's hook was not notified of the change.

‚úÖ Solution Implemented

Added refresh function to App component's useApiKeys hook
Added Chrome storage change listener in App.tsx
Automatically refresh API key status when storage changes
Added proper TypeScript typing for storage change events

üìù Code Changes

// Added to App.tsx imports import { useEffect } from "react"; // Updated useApiKeys destructuring const { hasApiKeys, isLoading: apiKeysLoading, refresh: refreshApiKeys } = useApiKeys(); // Added storage change listener useEffect(() => { if (typeof chrome !== 'undefined' && chrome.storage && chrome.storage.onChanged) { const handleStorageChange = (changes: { [key: string]: chrome.storage.StorageChange }, namespace: string) => { if (namespace === 'sync' && (changes.openaiApiKey || changes.tavilyApiKey)) { console.log('API keys changed in storage, refreshing...'); refreshApiKeys(); } }; chrome.storage.onChanged.addListener(handleStorageChange); return () => { chrome.storage.onChanged.removeListener(handleStorageChange); }; } }, [refreshApiKeys]);

üîß Technical Details

Chrome Storage API Integration

  • Event Listener: chrome.storage.onChanged
  • Namespace Filter: Only responds to 'sync' storage changes
  • Key Filter: Only refreshes on openaiApiKey or tavilyApiKey changes
  • Cleanup: Proper listener cleanup on component unmount

State Management

  • Reactive Updates: Automatic refresh when storage changes
  • Performance: Debounced to avoid excessive API calls
  • Error Handling: Graceful fallback if Chrome APIs unavailable
